# EduFinance

EduFinance — учебное full-stack веб-приложение для учета личных финансов, моделирования платежей, анализа доходов и расходов и демонстрации базовых принципов построения fintech-системы.

Проект подготовлен для производственной эксплуатационной практики по направлению 02.03.03 «Математическое обеспечение и администрирование информационных систем», профиль «Разработка программного обеспечения (Full-stack разработка)».

## Назначение проекта

Приложение позволяет пользователю:

- зарегистрироваться и войти в систему;
- создавать финансовые счета;
- фиксировать доходы и расходы;
- распределять операции по категориям;
- рассчитывать текущий баланс;
- имитировать платежную операцию без подключения к банковским API;
- просматривать простую аналитику по доходам, расходам и категориям;
- разграничивать доступ между пользователем и администратором.

В проекте не используются настоящие банковские карты, платежные реквизиты, паспортные данные и реальные финансовые API. Все данные условные и предназначены только для учебной проверки.

## Локальный запуск

1. Создать виртуальное окружение:

```bash
python -m venv .venv
```

2. Активировать окружение.

Для Windows:

```bash
.venv\Scripts\activate
```

Для macOS / Linux:

```bash
source .venv/bin/activate
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Создать локальный файл настроек:

```bash
cp .env.example .env
```

На Windows можно скопировать файл вручную или выполнить:

```bash
copy .env.example .env
```

5. Инициализировать базу данных и демонстрационные записи:

```bash
flask --app run.py init-db
```

6. Запустить приложение:

```bash
flask --app run.py run
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:5000
```

## Основные функции

### Авторизация и роли

Пользователь может зарегистрироваться, войти в систему и выйти из нее. Администратор имеет доступ к панели управления, где отображаются пользователи, счета и журнал действий.

### Финансовые счета

Пользователь создает счета с указанием названия, валюты и начального баланса. Баланс пересчитывается после добавления операций и подтвержденных платежей.

### Операции

Поддерживаются два типа операций: доход и расход. Каждая операция связана со счетом, категорией, датой, суммой и комментарием.

### Имитация платежей

Платеж проводится внутри учебной системы. Если средств на счете достаточно, платеж получает статус `confirmed`, а сумма списывается со счета через расходную операцию. Если средств недостаточно, платеж получает статус `declined`.

### Аналитика

Аналитический модуль показывает общий баланс, сумму доходов, сумму расходов и распределение расходов по категориям.

## Безопасность

В проекте реализованы базовые меры защиты:

- хеширование паролей;
- CSRF-защита форм;
- проверка входных данных;
- разграничение доступа по ролям;
- ограничение административных маршрутов;
- исключение секретных ключей из репозитория;
- хранение локальных параметров запуска в `.env`;
- журналирование значимых действий.

Перед публикацией проекта на GitHub нельзя добавлять файл `.env`, локальную базу данных, дампы с персональными данными, пароли, токены и секретные ключи.
